3.2 B

3.2.1 BELL Alarm Code


BELL-001 WARN Unknown BellTool error

Cause: BellTool has detected an unknown error.

Remedy: See TP error log for cause of error.


BELL-002 WARN Press master reset

Cause: System is in an error condition.

Remedy: Press reset to clear error.


BELL-003 WARN Reset failure

Cause: The reset procedure has failed.

Remedy: Correct the error, if any, and try the reset again.


BELL-004 WARN Machine 1 is bypassed

Cause: Bypass Machine 1 input signal is ON.

Remedy: The bypass machine 1 input must be OFF to enable the machine.


BELL-005 WARN Machine 2 is bypassed

Cause: Bypass Machine 2 input signal is ON.

Remedy: The bypass machine 2 input must be OFF to enable the machine.


BELL-006 WARN Servo disconnect switch off

Cause: Servo disconnect on machine is ON.

Remedy: Set servo disconnect OFF.


BELL-007 PAUS Auto mode disabled in cycle

Cause: During a cycle, the mode select input was switched from automatic.

Remedy: Status message only.


BELL-008 WARN Failed to run %s task

Cause: An attempt to run a BellTool task has failed.

Remedy: Cycle power on the controller.


BELL-009 WARN REMOTE switch must be REMOTE

Cause: Remote switch is in the LOCAL position.

Remedy: Place remote switch to REMOTE position.


BELL-010 WARN STEP mode must be disabled

Cause: STEP mode is ON going into production.

Remedy: Press TP STEP key to disable STEP mode.


BELL-011 WARN Machine lock is ON

Cause: Machine lock is ON going into production.

Remedy: Disable machine lock on TP.


BELL-012 WARN Machine motion is not enabled

Cause: UOP enable input is OFF.

Remedy: UOP enable input must be ON.


BELL-013 WARN Genoveride not 100 %

Cause: Program cannot set genoverride to 100.

Remedy: Manually set genoverride to 100 on TP.


BELL-014 WARN Waiting for machine to be home

Cause: Machines must be at home position to enter automatic mode.

Remedy: Send machine to the home position.


BELL-015 PAUS Machine 1 bypassed in cycle

Cause: During a cycle, the bypass switch for machine 1 was turned on.

Remedy: Status message only.


BELL-016 PAUS Machine 2 bypassed in cycle

Cause: During a cycle, the bypass switch for machine 2 was turned on.

Remedy: Status message only.


BELL-017 PAUS Step enabled during cycle

Cause: During a cycle, the step key was pressed on the teach pendant.

Remedy: Press TP STEP key to disable STEP mode.


BELL-018 WARN Top machine is bypassed

Cause: Bypass top machine input signal is ON.

Remedy: The bypass top machine input must be OFF to enable the machine.


BELL-019 PAUS Failed to continue motion task

Cause: An attempt to continue the motion task has failed.

Remedy: The current cycle must be aborted.


BELL-020 PAUS Failed to pause motion task

Cause: An attempt to pause the motion task has failed. Presumably, the motion task has aborted.

Remedy: The current cycle must be aborted.


BELL-021 PAUS Fast-Up/In motion

Cause: A fast-up move was requested by the cell controller.

Remedy: Correct the problem and reset the cell. Cancel or continue the current cycle, if any.


BELL-022 WARN Both side machines are bypassed

Cause: Both machine 1 and machine 2 bypass inputs are ON.

Remedy: At least one machine bypass input must be OFF to enable the machine and enter automatic mode.


BELL-023 PAUS Test cycle aborted

Cause: The test cycle was aborted

Remedy: Reset major fault condition.


BELL-024 WARN Brake release switch on

Cause: The brake release switch is on

Remedy: The brake release switch must be off to reset the controller.


BELL-025 WARN Beam out of balance

Cause: The beam is out of balance.

Remedy: Jog one of the vertical and/or tracking drives until the beam is in balance. The machine may require re-mastering.


BELL-026 WARN Conveyor simulation on

Cause: Conveyor simulation was turned on.

Remedy: Status message only.


BELL-027 WARN Conveyor simulation off

Cause: Conveyor simulation was turned off.

Remedy: Status message only.


BELL-029 WARN Oscillation axis overspeed

Cause: An oscillation axis overspeed condition could occur with the current oscillation values.

Remedy: Decrease the oscillation distance or increase the oscillation step size.


BELL-030 WARN Trigger %s in non-paint zone

Cause: A trigger point has been defined within a non-paint zone. Although this is a valid use of a trigger, a warning message is posted to warn the user of the potential error

Remedy: Review the trigger definitions.


BELL-031 WARN Non-increasing fluid zones

Cause: Fluid zone locations must be taught with increasing conveyor values.

Remedy: Re-teach the zone locations so that the associated conveyor values increase with each new fluid zone.


BELL-032 WARN Non-increasing trigger zones

Cause: Trigger zone locations must be taught with increasing conveyor values.

Remedy: Re-teach the trigger locations so that the associated on conveyor values increase with each new trigger.


BELL-033 WARN %s overspeed

Cause: The indicated axis will exceed it’s maximum speed limit.

Remedy: Increase the conveyor position relative to the previous points conveyor position or decrease the change in axis position relative to the previous points axis position. In the case of a top machine, the maximum conveyor speed can be reduced.


BELL-034 WARN Can’t find min pnt spd

Cause: The minimum paint speed could not be determined because an axis overspeed condition occurred.

Remedy: Modify the path positional parameters and/or reduce the maximum conveyor speed


BELL-035 WARN Can’t optimize max pnt spd

Cause: The path could not be fully optimized for maximum paint speed because an axis overspeed condition occurred

Remedy: Modify the path positional parameters and/or reduce the maximum conveyor speed


BELL-036 WARN Purge position is undefined

Cause: The purge position has not been taught.

Remedy: Use the Move Menu to teach the purge position.


BELL-037 PAUS Aux Position %s is undefined

Cause: An auxiliary position is not taught.

Remedy: Use the Move menu to teach the auxiliary position.


BELL-038 WARN Encoder count rollover

Cause: The encoder counter has detected a rollover condition.

Remedy: This is a status/warning message only.


BELL-039 PAUS Bad tracking hardware

Cause: Bad tracking sensor hardware interface.

Remedy: Check all sensor hardware, cables, and connections.


BELL-040 PAUS Illegal encoder multiplier

Cause: Illegal encoder multiplier number.

Remedy: Use a valid encoder multiplier number.


BELL-041 PAUS Encoder not enabled

Cause: Tracking encoder is not enabled.

Remedy: Enable the tracking encoder before readings its count within the program


BELL-042 PAUS Encoder is disconnected

Cause: Controller is detecting a problem with the encoder cable.

Remedy: Check encoder cable.


BELL-043 WARN Trigger zone %s non-increasing

Cause: Each trigger zone must be taught with increasing conveyor values.

Remedy: Re-teach the trigger locations so that the associated conveyor values increase within each trigger zone.


BELL-044 PAUS Motion task has faulted

Cause: The motion task has faulted, but was not detected by the main task.

Remedy: Current cycle must be aborted.


BELL-045 PAUS Failed to resume cycle

Cause: The motion task failed to resume the current cycle.

Remedy: Current cycle must be aborted.


BELL-046 PAUS Dual Drive adjust mode selected

Cause: Dual drive adjust mode has been selected at the RJ controller.

Remedy: Cancel adjust mode at the SYSTEM Variables menu. Master and calibrate the machine.


BELL-047 WARN BellWorks not reading changes

Cause: BellWorks is not receiving the data changed at the RJ controller

Remedy: Verify that all job changes have been uploaded to BellWorks. Also, verify that communications between the PLC and the RJ controller are ok.


BELL-048 PAUS Unknown job init from PLC

Cause: The PLC has sent an unknown job number init to the RJ controller

Remedy: Verify that all jobs have been loaded on the RJ controller. Also, verify that communications between the PLC and the RJ controller are ok.


BELL-049 WARN Error reading BellWorks Data

Cause: Could not read a program file that was transferred from BellWorks

Remedy: Check communications between BellWorks and the RJ controller


BELL-050 PAUS Job data not processed

Cause: Program data has not been processed

Remedy: Verify that all programs copied from BellWorks to RJ are processed


BELL-051 PAUS PLC e-stop detected

Cause: An interlock signal fault or PLC E-stop was detected.

Remedy: Clear source of fault.


BELL-052 PAUS Machine overtravel

Cause: The mechanical unit went beyond its normal travel limits and reached an overtravel limit switch.

Remedy: Follow the overtravel recovery procedure in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-053 PAUS Release SOP e-stop

Cause: The operator panel (SOP) E-STOP has been detected.

Remedy: Turn/Release the SOP E-STOP button to clear the E-STOP condition.


BELL-054 PAUS Release TP e-stop

Cause: The teach pendant E-STOP has been detected.

Remedy: Push in teach pendant E-STOP button again to clear the E-STOP condition.


BELL-055 PAUS TP enabled during production

Cause: The teach pendant cannot be enabled during production.

Remedy: Disable the teach pendant.


BELL-056 PAUS Disable teach pendant

Cause: Teach pendant is enabled.

Remedy: Disable the teach pendant to continue the program.


BELL-057 PAUS Stop error excess

Cause: The servo error was too big when the machine had stopped.

Remedy: Refer to troubleshooting section in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-058 PAUS Move error excess

Cause: The servo error was too big when the machine was moving.

Remedy: Refer to troubleshooting section in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-059 PAUS DEADMAN switch RELEASED

Cause: The DEADMAN switch has been released during an operation. The operation has been discontinued.

Remedy: Press the DEADMAN switch, and hold it down to restart the procedure.


BELL-060 PAUS Fence open during production

Cause: FENCE1 and FENCE2 circuit open on EMG control board.

Remedy: Determine the cause of FENCE1 and FENCE2 open circuit and correct.


BELL-061 PAUS UOP immediate stop detected

Cause: IMSTP (immediate stop) UOP input not asserted.

Remedy: If using UOP, clear source of fault. If not using UOP, select the I/O menus, zero UOP mapping, and cycle power on the controller.


BELL-062 PAUS HOLD active

Cause: A HOLD signal, either from the UOP or the teach pendant, has been activated.

Remedy: Remove the HOLD condition


BELL-063 PAUS PC serial interface fault

Cause: PC interface serial link has failed.

Remedy: Check LED status on PC interface board. Refer to manual for possible cause.


BELL-065 PAUS OVC alarm

Cause: The average current calculated by the servo software exceeded specification.

Remedy: Refer to troubleshooting section in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-066 PAUS Machine not calibrated

Cause: The machine must be calibrated before proceeding. Only jogging in the joint coordinate system and the calibration motion itself can be executed before the machine is calibrated.

Remedy: Calibrate the machine.


BELL-067 PAUS Limit error

Cause: The taught position was outside the normal work envelope of the robot. One of the machine axes could not reach the position.

Remedy: Reteach the position.


BELL-068 PAUS Position not reachable

Cause: There is no solution (conversion from a Cartesian position to joint angles) for the taught position.

Remedy: Reteach the position.


BELL-069 PAUS Configuration mismatch

Cause: This is a pause condition. The configuration cannot change during a Cartesian (linear or circular) motion.

Remedy: Either reteach the motions using the same configuration or use a joint motion to change configurations before proceeding with Cartesian motion.


BELL-070 PAUS Planner error

Cause: An error was encountered by the planner.

Remedy: Check the error message log for details.


BELL-071 PAUS Uninitialized position

Cause: This is a system software error.

Remedy: Document the events that led to the error, record the software version number and contact your local distributor or the Fanuc hotline. You can try to clear the error by powering the controller off and then on.


BELL-072 PAUS Brake fuse blown

Cause: The brake fuse is blown on the EMG control pcb. The FALM light on the EMG control pcb should be light.

Remedy: Refer to troubleshooting section in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-073 WARN Machine not mastered

Cause: Machine is not mastered.

Remedy: Refer to mastering section in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-074 PAUS MCC alarm

Cause: The servo amplifier magnetic contactor (MCC) is welded closed.

Remedy: Refer to troubleshooting section in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-075 PAUS Discharge current alarm

Cause: The regenerative energy produced by the motor exceeded specifications.

Remedy: Refer to troubleshooting section in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-076 PAUS High voltage alarm

Cause: The DC voltage on the main power circuit of the servo amplifier exceeded specification.

Remedy: Refer to troubleshooting section in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-077 PAUS High current alarm

Cause: The current in the main power circuit of the servo amplifier exceeded specification.

Remedy: Refer to troubleshooting section in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-078 PAUS Low voltage alarm

Cause: The DC voltage on the main power circuit of the servo amplifier is lower than the specification even though MCC is on.

Remedy: Refer to troubleshooting section in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-079 PAUS Servo amp overheat alarm

Cause: The servo amplifier overheated.

Remedy: Refer to troubleshooting section in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-080 PAUS Collision alarm

Cause: The servo software detected a disturbance torque that was too high, and tripped a a collision detection alarm.

Remedy: Refer to troubleshooting section in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-081 WARN Pulse coder battery discon alarm

Cause: The battery voltage for the pulse coder is zero volts.

Remedy: Refer to troubleshooting section in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-082 PAUS Pulse coder alarm

Cause: Pulse coder alarm had occurred. Go to the ALARMS screen on the teach pendant to record the type of pulse coder alarm.

Remedy: Refer to troubleshooting section in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-083 WARN Pulse coder battery low alarm

Cause: The pulse coder batteries are low.

Remedy: Refer to troubleshooting section in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-084 PAUS Motor overheat occurred

Cause: The pulse coder overheated.

Remedy: Refer to troubleshooting section in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-085 PAUS Pulse coder communication error

Cause: The axis control pcb sent the request signal, but did not receive serial data from the pulse coder.

Remedy: Refer to troubleshooting section in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-086 PAUS Pulse coder position compare error

Cause: The feedback velocity exceeds specification.

Remedy: Refer to troubleshooting section in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-087 PAUS Collision Detection (Hand Broke)

Cause: Collision Detection (Hand Broke)

Remedy: Refer to troubleshooting section in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-088 PAUS CRCERR alarm

Cause: Serial data from pulse coder changed during communication to axis control pcb.

Remedy: Refer to troubleshooting section in the SYSTEM R-J Controller Series Electrical Connection and Maintenance Manual.


BELL-089 PAUS TP disabled during motion

Cause: The teach pendant was disabled during a motion issued from the teach pendant.

Remedy: The teach pendant must remain enabled during motions issued from the teach pendant.


BELL-090 WARN Cannot calibrate when bypassed

Cause: An attempt was made to calibrate when the machines are bypassed.

Remedy: Unbypass the machines and rerun the auto calibration procedure.


BELL-091 WARN %s trig too early

Cause: The conveyor position for the trigger zone is before the path is started.

Remedy: Enter a conveyor position after the path start position.


BELL-092 WARN %s fluid too early

Cause: The conveyor position for the fluid zone is before the path is started.

Remedy: Enter a conveyor position after the path start position.


BELL-093 PAUS Reverse conv. during cycle

Cause: The conveyor has traveled in reverse greater than allowed during a cycle.

Remedy: Reset the major fault and cancel or continue the cycle.


BELL-094 PAUS Axis %s DJ delta too large

Cause: The motion command is too large and would cause an axis overspeed condition.

Remedy: Re-teach/save the path for the desired conveyor speed.


BELL-095 WARN %s limit error

Cause: The indicated axis cannot reach the programmed position without exceeding a joint limit.

Remedy: Make sure the programmed position is within the joint limits of the machine. If the job contains oscillation motion, make sure the oscillation deltas, when applied to the nominal positions, do not exceed the limits.


BELL-096 WARN Osc zone %s superseded

Cause: The indicated oscillation zone is superseded by a previous zone.

Remedy: Oscillation zone transitions always occur when the oscillation axis is at its zero position. This means that when the machine reaches a position boundary, oscillation may continue based on the previous positions oscillation parameters until the oscillation axis reaches its zero position. If positions are taught very close together, oscillation zones may be superceded by a previous zone


BELL-101 WARN Position may not be cleared

Cause: An attempt was made to clear one of the pre-defined auxiliary positions.

Remedy: Do not attempt to clear these positions. They are necessary for proper operation of the system.


BELL-102 WARN Not in teach mode

Cause: An attempt was made to perform a manual operation from outside of teach mode.

Remedy: Certain operations are only permitted in teach mode. Do not attempt to perform this operation in any other mode.


BELL-103 WARN Can’t overwrite processed data

Cause: An attempt was made to store an un-processed job, overwriting an processed job. This operation is prohibited.

Remedy: Modify the job data so that the processing step completes successfully. Also, an un-proceesed job may be stored with a new name, using the STOR-AS function.


BELL-104 WARN Too many positions in job

Cause: The current job already contains the maximum number of positions. No more may be added until others are first deleted.

Remedy: Delete any unnecessary positions.


BELL-105 WARN No positions in job

Cause: The current job contains no positions. None may be deleted.

Remedy: Do not attempt to delete positions from a job until some have been added.


BELL-106 WARN Job does not exist

Cause: The requested job does not exist.

Remedy: Select a different job for this operation.


BELL-107 WARN 1st posn must be non-paint

Cause: The first position in the job has the paint mode set true. Motion to the first position must always be non-paint motion.

Remedy: Change the paint mode of the first position to OFF.


BELL-108 WARN 2nd posn must be paint-on

Cause: The second position in the job has the paint mode set false. Motion to the second position must always be a paint motion.

Remedy: Change the paint mode of the second position to ON.


BELL-109 WARN Non-increasing conveyor values

Cause: The job contains a zone of zero size.

Remedy: Modify the conveyor values so that the value at each position is greater than at the previous position.


BELL-110 WARN Max conveyor speed is too small

Cause: The max conveyor speed is less than or equal to zero. This is too small.

Remedy: Increase the max conveyor speed parameter to a value greater than zero.


BELL-111 WARN Press shift-reset & hold shift

Cause: System is in an overtravel condition.

Remedy: Press shift-reset to clear overtravel, then hold shift key and jog machine off the overtravel.


BELL-112 WARN Minimum %s positions to process

Cause: An attempt was made to process a job with less than 2 positions.

Remedy: Do not attempt to process a job until some positions have been added.


BELL-120 WARN I/O Hardware not installed

Cause: I/O Hardware selected is physically not installed

Remedy: Make sure I/O Hardware that is being selected is installed correctly!


BELL-121 WARN Host name not recognized

Cause: Hostname currently setup on the controller for ethernet communications is not per convention for automatic Host Comm setup.

Remedy: If standard BellTool Ethernet communications setup is desired, enter the correct ’Network Setup Number’ at Setup screen at Controlled Start and press [F2] Setup to reconfigure. If you do NOT want Ethernet to be setup automatically, ignore this message and leave ’Network Setup Number’ at 0.
